SetAnsiCodepage (sub)

Will set the given ANSI codepage and do the required unicode remapping. This is not required for the various _UPRINT functions, but is useful to have the correct charset for the regular PRINT/_PRINTSTRING functions. Best is the same codepage, which you've set in your QB64 IDE.

SYNTAX:

SetAnsiCodepage cpName$

INPUTS:

cpName$ (STRING)

  • The desired codepage name as per the list below, upper/lower case doesn't matter. If you give an invalid name, then the current unicode mapping remains unchanged.
    • Pc437, Pc737, Pc775, Pc850, Pc852, Pc855, Pc857, Pc860, Pc861, Pc862, Pc863, Pc864, Pc865, Pc866, PcMIK, Pc869, Pc874, Win1250, Win1251, Win1252, Win1253, Win1254, Win1255, Win1256, Win1257, Win1258
